# Save Progress Skill

작업 완료 후 Git 저장(커밋+푸시), 프로젝트 문서 업데이트, 교훈 기록을 간결하게 처리하는 스킬입니다.

**핵심 원칙: 필요한 것만, 간결하게**
- Git 커밋 + 푸시
- progress.md만 업데이트 (날짜별 섹션)
- CLAUDE.md에 "다음 세션 가이드" 추가
- 글로벌 교훈 제안
- **불필요한 파일 생성 금지** (README, work-summary, commit-message.txt 등)

## 실행 순서

### 1. Git 변경사항 분석 및 커밋

먼저 현재 변경사항을 확인:

```bash
git status
git diff
```

변경된 파일들을 분석하여 **의미 있는 커밋 메시지**를 생성합니다. 커밋 메시지는 다음 형식을 따릅니다:

```
<제목: 50자 이내로 변경사항 요약>

주요 변경사항:
- 변경 1
- 변경 2
- 변경 3

Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

**중요**: 커밋 메시지를 생성한 후 사용자에게 보여주고 승인을 받습니다:

```
이 커밋 메시지로 진행할까요?

[생성된 커밋 메시지]

(y: 승인 / n: 취소 / 수정 내용 입력)
```

승인 받은 후 **반드시 푸시까지** 수행:

```bash
git add <변경된 파일들>
git commit -m "$(cat <<'EOF'
<커밋 메시지>
EOF
)"
git push origin <현재 브랜치>
```

**중요**: Git push까지 완료해야 작업이 끝난 것입니다.

### 2. 프로젝트 문서 업데이트

#### progress.md 업데이트

`progress.md` 파일이 존재하는지 확인하고, 없으면 생성합니다. 날짜별로 진행 내용을 추가:

```markdown
# 프로젝트 진행 현황

## YYYY-MM-DD

### 완료
- [x] 완료된 작업 1
- [x] 완료된 작업 2

### 진행 중
- [ ] 진행 중인 작업 1

### 다음 작업
- [ ] 예정된 작업 1

### 주요 변경사항
- 변경사항 요약 1
- 변경사항 요약 2

### 메모
- 특이사항이나 주의할 점
```

**작성 방법**:
1. 오늘 날짜 섹션이 이미 있으면 업데이트
2. 없으면 맨 위에 새로 추가
3. 세션 대화 내용에서 완료된 작업, 진행 중인 작업, 주요 변경사항을 추출
4. 사용자가 언급한 다음 계획이 있으면 "다음 작업"에 추가

#### CLAUDE.md 업데이트

프로젝트의 `CLAUDE.md` 파일을 업데이트합니다:

1. **현재 상태 섹션 업데이트**: 프로젝트의 현재 상태 반영
2. **다음 세션 가이드 추가/업데이트**:
   ```markdown
   ## 다음 세션 시작 시

   **필수 확인**:
   - `progress.md` 읽고 현재 진행 상황 파악
   - 최근 커밋 로그 확인 (`git log --oneline -5`)

   **현재 작업 중**:
   - [작업 내용]

   **다음 할 일**:
   - [ ] 작업 1
   - [ ] 작업 2
   ```

3. **중요 설정/환경 변경사항**: 새로 추가된 환경변수, 설정 파일, API 키 등이 있으면 문서화

### 3. 글로벌 교훈 제안

세션 대화 내용을 분석하여 **재사용 가능한 패턴이나 교훈**을 추출합니다.

#### 3-1. 중요도 필터링

**높은 중요도만 추출** (이것들만 제안):
- ✅ 실제로 버그나 문제를 일으켰던 교훈
- ✅ 비용/시간 낭비를 크게 줄일 수 있는 것
- ✅ 여러 프로젝트에서 반복적으로 적용 가능한 것

**낮은 중요도는 스킵** (제안하지 않음):
- ❌ 특정 프로젝트에만 해당하는 내용
- ❌ 상식적인 내용 ("코드 작성 전에 생각하기" 같은)
- ❌ 한 번만 겪을 법한 특수한 상황

#### 3-2. 중복 체크

글로벌 CLAUDE.md(`~/.claude/CLAUDE.md`)를 읽고 **맥락/의도 기반으로 중복 확인**:

1. 제안하려는 교훈의 핵심 의도 파악
2. 글로벌 CLAUDE.md의 각 섹션과 의도/맥락 비교
3. 판단:
   - 이미 비슷한 의도를 다루고 있음 → **스킵**
   - 단어는 비슷하지만 맥락이 다름 → **제안**
   - 완전히 새로운 교훈 → **제안**

**중요**: 키워드 매칭이 아니라 **의미 기반 판단**할 것

**예시**:
- 제안: "요구사항 명확화 우선"
- 글로벌: "모호함을 줄이기 위해 사전에 상세 스펙 작성"
- 판단: 둘 다 "작업 전 요구사항 명확히"라는 같은 의도 → **중복, 스킵**

#### 3-3. 사용자에게 제안

중요도 필터링 + 중복 체크를 통과한 교훈만 제안:

```
다음 내용을 글로벌 CLAUDE.md에 추가할까요?

1. [교훈 1 제목]
   내용 요약...

2. [교훈 2 제목]
   내용 요약...

추가할 항목 번호를 선택하세요 (예: 1,2 또는 all 또는 none):
```

선택된 교훈을 `~/.claude/CLAUDE.md`의 적절한 섹션에 추가합니다.

### 4. 완료 보고

**간결하게** 보고:

```
✅ 저장 완료!

Git: 커밋 & 푸시 완료 (<커밋 해시>)
progress.md: 2026-03-02 작업 기록
CLAUDE.md: 다음 세션 가이드 업데이트
글로벌 교훈: <N>개 추가
```

**금지사항**:
- 불필요한 파일 생성 금지 (README.md, work-summary.txt, commit-message.txt 등)
- 장황한 보고서 작성 금지
- 이미 존재하는 파일만 업데이트

## 에러 처리

### Git 푸시 실패

#### 원격 저장소가 없는 경우
```bash
git push origin master
# fatal: 'origin' does not appear to be a git repository
```

**처리 방법**:
1. 사용자에게 명확히 알림:
   ```
   ❌ GitHub 저장소가 없습니다.

   GitHub 저장소를 생성할까요?
   - y: GitHub 저장소 자동 생성 (gh repo create 사용)
   - n: 푸시 건너뜀 (로컬 커밋만 완료됨)
   ```

2. 사용자 선택에 따라:
   - **y**: `gh repo create` 실행 후 푸시 재시도
   - **n**: "✅ 로컬 커밋 완료 (푸시는 나중에 수동으로 진행하세요)"

#### 원격 브랜치와 충돌
- 사용자에게 알리고 `git pull` 제안

#### 권한 문제
- 인증 상태 확인 요청

### 파일 쓰기 실패
- 권한 문제: 파일 경로 확인 및 권한 요청
- 파일 잠김: 다른 프로그램에서 파일을 열고 있는지 확인

### 변경사항 없음
Git에 변경사항이 없으면 문서 업데이트만 진행하고 사용자에게 알림

## 주의사항

- 항상 사용자 확인을 받은 후 Git 작업 수행
- 중요한 파일(CLAUDE.md, progress.md)을 덮어쓰기 전에 현재 내용 확인
- 글로벌 CLAUDE.md는 기존 내용을 유지하면서 새 내용만 추가
- 커밋 메시지는 구체적이고 의미 있게 작성
